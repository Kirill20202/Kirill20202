Сборка своей Windows

Существуют определенные сайты для создания Windows под себя из коробки, а точнее создание .xml файла, который уже редактирует установку системы.В этой статье поэтапно разберем создание своего .xml файла. Далее будут описания и скрины каждого параметра.

Сайт Schneegans и описание с самого сайта:
Эта служба позволяет создавать файлы ответов (обычно называемые Unattend.xml или autounattend.xml) для выполнения Untairdendend Suppartions установки Windows 10 и Windows 11, включая 24H2. Файлы ответов, сгенерированные этой службой, в первую очередь предназначены для использования с установкой Windows, выполненной от Windows PE для выполнения чистых (а не обновления) установки.

На сайт можно загрузить чужой .xml файл и посмотреть его конфигурацию. Можно поставить стандартную конфигурацию или минимально возможную (конечно советую собирать самому под своми нужды).

Следующий параметр это выбор языковых настроек, советую сразу скачивать Windows с нужным языком, а на сайте для более быстрой установки выбирайте первый пункт и нужный язык.

Здесь вы выбираетете свою архитектуру процессора, удерживая shift можно выбрать несколько архитектур для универсальной установки.

Первый чекбокс это проверка требований сисемы, его лучше включать, система установиться бысрее и подойдет под разные системные характеристики. Второй чекбокс лучше не включать, он отвечает за установку Windows без интернета. Третий чекбокс будет копировать файл .xml в целевой раздел загрузки при установке из под системы, что я не советую, лучше устанавливаться в чистую с флешки. И последний флажок отвечат за скрытие окон Powershell при установке системы, когда сама Windows начнёт установку будут появляться окна Powershell, которые нельзя закрывать, если они вам не мешают и вы их точно не закроете, то можете не ставить этот параметр.

Вы можете выбрать случайное имя компьютера, использовать своё или написать скрипт Powershell, который по вашим условиям будет создавать имя компьютеру

Следующий параметр это выбор ОС. Оставить выбор установки Compact OS на Windows, установить Compact OS принудительно или не устанавливать Compact OS
Compact OS - позволяет запускать операционную систему из сжатых файлов. Однострадирование помогает запускать предварительно загруженные классические приложения Windows в сжатых файлах. К сожелению Compact OS иногда может устанавливаться некоректно, поэтому советую её не устанавливать или дать выбор Windows

Выбор часового пояса:

  • Позволить Windows определить часовой пояс на основе языка и региона
  • Установить пояс вручную

Разделы и форматирование диска

  • Во время установки Windows, вручную выбрать диск для установки
  • Windows автоматически очистит, разобьет и отформатирует диск с помощью выбраных настроек:
    Выбор схемы разделов: GPT - современная схема разбиения дисков, основная система запуска это UEFI, нужно выбрать размер ESP файла, файла загрузки системы и компонентов, можно оставить около 300 МБ
    MBR - старая, но довольно популярная схема разделов, используется в старых BIOS
    Среда восстановления Windows (WinRE) - среда восстановления, может исправить некоторые ошибки при запуске или помочь загрузиться с USB/CD

    • Варианты установки:

    Создайте отдельный раздел с выбранным размером и установите на него Windows RE.
    Установить на раздел Windows, раздел восстановления не будет создан.
    Удалить папку C:\Recovery и, таким образом, освободить около 600 МБ места на диске, раздел восстановления не будет создан.

  • Использовать diskpart скрипты для конфигурации диска.
    Это скрипты для автоматизирующих задач связанных с дисками, ниже приведен пример скрипта и подсказка:
    Если ваш диск уже разбит на разделы и отформатирован, введите команду diskpart, которая не даст никакого эффекта, например REM или SELECT DISK=0. Также обратите внимание, что назначения букв дисков (например, ASSIGN LETTER=R) не сохранятся.

    Выберите раздел для установки Windows после запуска скрипта:
    Установите Windows на первый доступный раздел, в котором достаточно места и который еще не содержит установку Windows
    Установить на другой выбранный раздел.

  • Можно запустить VBScript скрипт для проверки вашего diskpart скрипта или неожиданных индексных номеров.

Активация Windows

  • Использовать универсальный ключ продукта, такой ключ можно использовать для установки Windows, но он не активирует ее.
  • Использовать свой ключ продукта
  • Если не доверяете сайту можете в будущем сами поменять ключ уже в готовом .xml файле
  • Использовать ключ продукта, сохраненный в прошивке BIOS/UEFI, если на вашем компьютере была предустановлена Windows

Профиль Windows

Вы можете предвварительно создать локальные профили, для правильного применения настроек нужно чтобы Windows загрузилась от пользователя администратора:

  • Войти в учётную запись созданную выше
  • Войти во встроенную учётную запись системы и установить пароль
  • Не входить в систему(не рекомендуется)

Чекбокс отвечает за скрытие паролей в файле системой Base64

Первый вариант войти в онлайн учётную запись во ремя установки

Второй вариант войти в локальную учётную запись во время установки

Пароль в Windows (локальной учётной записи) имеет свойство истекать, во избежание этого можно поставить первый параметр:

  • Пароль не истекает
  • Стандартные настройки Windows, пароль истекает через 42 дня
  • Истекает через выбранное время

Windows может заблокировать учётную запись после 10 неудачных попыток ввода пароля, это называется Lockout policy:

  • Использовать стандартные настройки
  • Отключить политику, уязвим для попытки взлома брутфорсом
  • использовать свои выбранные параметры

Вы можете настроить показ скрытых и системных файлов проводника

  • Стандартные настройки, скрывать скрытые файлы
  • Скрывать скрытые и системные файлы
  • Показывать все файлы

Чекбоксы отвечают за:

Всегда показывать расширения файлов, по умолчанию Windows скрывает расширения

Использовать классическое контекстное меню

Открывать файловый менеджер вместо быстрого доступа

Показывать кнопку "Завершить задачу" на панели задач

Моя версия:

Далее идёт настройка внешнего вида системы, в первом окне выбирается вид значка Windows и панели задач.

При первом входе в систему на панеле задач будут закреплены программы, такие как, Проводник, Microsoft Edge, Store, что бы избавиться от них во втором окне можно оставить стандартные закрепления, удалить все, или самому написать какие программы будут закреплены.

Чекбоксы отвечают за:

  • Отключение виджетов, уберёт новости и погоду в левом нижнем углу
  • Сместить панель задач Windows к левому краю
  • Скрыть кнопку "Просмотр задач"
  • Всегда показывать все значки в трее, по стандарту Windows может скрывать некоторые программы
  • Не показывать результаты Bing при поиске в меню "Пуск"

Следующее окно это настройка плиток в Windows 10, можно оставить стандартные, удалить все, или создать .xml файл с определёнными плитками.

И последнее окно это приложения в меню пуск, которые также можно оставить по стандарту(не рекомендуется, есть риск возникновения конфликта и иконки всё равно будут добавлены в меню), удалить все и создать свой json файл с настройкой меню.

Далее идут настройки системы, описание каждого чекбокса:

  • Перманентное отключение Windows Defender
  • Отключить обновления в Центре обновления Windows
  • Отключение User Account Control, это окна с подтверждением скачивания или разрешения администратора
  • Отключение Smart App Control - это проверка безопастности файлов и программ перед их запуском, если отключить то включить его снова уже нельзя
  • Отключение SmartScreen в Windows b Microsoft Edge, это окна с предупреждением "Система Windows защитила ваш компьютер, неопознаное приложение могло подвергнуть компьютер опастности"
  • Отключение Fast Startup - этот параметр заставляет систему запускаться быстрее, но есть минусы: если нужна полная перезагрузка для обновления это невозможно, нет доступа к файлам из другой ОС, нельзя попасть в BIOS/UEFI
  • Windows по умолчанию создаёт точки восстановления(сохранение системы), чтобы при небходимости откатиться на эту точку, тут можно отключить эту настройку и освободить часть места на диске
  • Включить длинные пути, это увеличит эффективность системы и программ
  • Включить Remote Desktop services - это технология, которая позволяет пользователям подключаться к рабочему столу и программам удаленного компьютера через сетевое соединение
  • Harden ACLs - это инструмент для управления доступом пользователей к папкам и файлам, управляемый администратором
  • Разрешить выполнение файлов сценариев PowerShell, политики выполнения определяют, можете ли вы загружать файлы конфигурации, такие как ваш профиль PowerShell, или запускать скрипты, разрешает выполнение неподписанных файлов .ps1
  • Не обновлять отметку времени последнего доступа, это увеличит производительность системы(система не будет записывать в журнал данные последнего доступа)
  • Запретить Центру обновления Windows перезагружать компьютер
  • Отключить системные звуки
  • Отключить предложения приложений Content Delivery Manager - это приложение которое может без вашего ведома устанавливать пакеты или программы(даже небезопасные). Обратите внимание, что в качестве побочного эффекта это также отключит кнопку "Управление устройствами" в разделе "Мобильные устройства" в "Настройках"
  • Запретить шифрование устройства, в противном случае Windows 11 автоматически включила бы шифрование BitLocker - если ваше устройство будет утеряно, то ваши данные на жёстком диске будут зашифрованы BitLocker и не доступны без ключа
  • Скрыть Edge First Run Experience, это позволит пропустить раздражающие диалоговые окна при первом запуске Edge
  • Отключить Edge Startup Boost, это остановит постоянную работу Edge в фоновом режиме
  • Удалить Microsoft Edge, Центр обновления Windows в конечном итоге может вернуть файл в исходное состояние, есть вариант переименовать файл C:\Windows\System32\IntegratedServicesRegionPolicySet.json в исходное состояние(сразу после установки системы)
  • Отключите параметр мыши Enhance Pointer Precision, это может быть полезно для игроков в шутеры от первого лица.
  • Удалить пустую папку C:\Windows.old, обычно при переустановке Windows из программы установки может создаваться эта папка со старыми данными
  • Включить аудит событий создания процессов, это инструмент позволяющий видеть неполадки в системе

Следующий параметр это визуальные эффекты (перемещение окон, сглаживание шрифта, тени и тд.):

Можно оставить стандартные значения Windows, настроить для улучшения внешнего вида, настроить для быстродействия системы и выбрать настройки самому

Далее, ярлыки на рабочем столе при первом запуске, первый вариант это стандартные ярлыки Windows, второй это выбрать ярлыки самому

В следущем параметре можно предустановить виртуальные машины и помощники к ним

  • Установить Oracle VirtualBox Guest Additions - это набор системных драйверов и приложений, которые оптимизируют производительность гостевых виртуальных машин и делают работу с ними удобнее
  • Установить VMware Tools - оптимизирует производительность и удобство использования
  • Установить VirtIO Guest Tools and QEMU Guest Agent - это вспомогательная программа для обмена информацией между хостом и гостем, также можно выполнять команды в гостевой системе

Подключение к Wi-Fi, можно выбрать в процессе установки, пропустить подключение, подключиться по введённым параметрам и настроить подключение в .xml файле

  • Включить отправку
  • Выбрать при установке
